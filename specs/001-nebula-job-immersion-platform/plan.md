# Implementation Plan: Nebula Job Immersion Program Platform

**Branch**: `001-nebula-job-immersion-platform` | **Date**: 2026-07-04 | **Spec**: [spec.md](spec.md)

---

## Summary

Build a full-stack Next.js (App Router) web application that allows coaches to create and manage Job Immersion Programs, students to discover and enroll in cohorts, and admins to monitor platform activity. The application uses a real NextAuth session backed by SQLite via Prisma, with a simplified Profile-based RBAC model. All business rules (enrollment capacity, status transitions, session scheduling constraints) are enforced server-side via Zod-validated server actions. The architecture is Feature-Driven — each domain is a self-contained module.

## Implementation Phases

### Phase 0 — Foundation
*Everything compiles, DB is live, auth works, RBAC is functional.*

| Task | Scope |
|---|---|
| Project scaffold | `create-next-app` + TypeScript strict + Tailwind |
| Prisma schema | Full schema from data-model.md; initial migration |
| PrismaClient singleton | `lib/prisma.ts` |
| NextAuth config | `lib/auth.ts` — CredentialsProvider, Prisma adapter, jwt/session callbacks with RBAC enrichment |
| Permission helpers | `lib/permissions.ts` — `hasPermission()`, `requirePermission()` |
| Logger | `lib/logger.ts` |
| Middleware | `middleware.ts` — route group protection |
| Seed script | Profiles, permissions, 3 test accounts, 1 sample published program + cohort |
| Dev Role Switcher | `DevRoleSwitcher.tsx` (NODE_ENV guard) |
| Shared UI primitives | Button, Input, Select, Badge, Card, Toast, Skeleton |
| Session scheduling util | `lib/session-scheduling.ts` |

### Phase 1 — Auth Feature
| Task | Scope |
|---|---|
| `/register` page | Public; auto-assigns Student profile; bcryptjs hash |
| `/login` page | CredentialsProvider form |
| Admin user creation | `createUser` action behind `user:create` permission |
| Unit tests | Schema validation, duplicate email, password hashing |

### Phase 2 — Programs Feature
| Task | Scope |
|---|---|
| Server actions | `createProgram`, `updateProgram` (with status guards), `getPublishedPrograms`, `getCoachPrograms` |
| Program catalog | `/programs` — cards, domain filter, search |
| Program detail | `/programs/[id]` — details + cohort list |
| Coach program list | `/coach/programs` — status badges |
| Create/Edit form | react-hook-form + zodResolver |
| Unit tests | Status transition logic, sessionCount lock |

### Phase 3 — Cohorts & Sessions Feature
| Task | Scope |
|---|---|
| Server actions | `createCohort`, `updateCohortStatus`, `suggestSessionDates`, `getCohortDetail` |
| Cohort creation form | Session scheduler UI with suggest-and-edit |
| Coach cohort detail | `/coach/cohorts/[id]` — enrolled students list |
| Unit tests | Session count guard, date range validation, auto-suggest algorithm |

### Phase 4 — Enrollment Feature
| Task | Scope |
|---|---|
| `enrollInCohort` action | `$transaction` + capacity check + auto-FULL transition |
| `getMyEnrollments` action | Full cohort details per enrolled student |
| Enroll button | Error states: full, closed, already enrolled |
| My Programs page | `/my-programs` — session schedule + explorations |
| Unit tests | Capacity edge cases, duplicate rejection |
| Playwright E2E | Full enrollment flow |

### Phase 5 — Explorations Feature
| Task | Scope |
|---|---|
| `createExploration` action | Coach ownership check |
| `getExplorationsForEnrolled` action | Enrollment-gated |
| Student exploration view | Listed under cohort in My Programs |
| Coach exploration form | Program or session link |
| *Bonus*: submit + feedback | `ExplorationSubmission` CRUD |
| Unit tests | Ownership and enrollment checks |

### Phase 6 — Admin Feature
| Task | Scope |
|---|---|
| `getDashboardStats` action | Aggregate queries; active = startDate ≤ today ≤ endDate |
| `getAllCohorts` action | Period classification (UPCOMING/ACTIVE/PAST) |
| `listUsers` + `createUser` | User management |
| Dashboard page | Stat cards + upcoming sessions + latest enrollments |
| Full cohorts view | `/admin/cohorts` — sortable table |
| User management | `/admin/users` |
| Unit tests | Period classification, stat accuracy |
| Playwright E2E | Dashboard access control, permission boundary |

### Phase 7 — Polish & Deliverables
